**分镜图片制作智能体：移动端 UI 设计规范**

**1.** **概述与目标**

本规范旨在为 “分镜图片制作智能体” 构建一套专业、精致且可落地的移动端（iOS/Android）UI 设计系统。设计核心在于摒弃传统的桌面端水平时间轴画布，转而采用更符合移动端操作习惯的**页面级流式设计**，以提升易用性、统一视觉风格并减少用户操作负担。

**2.** **设计令牌 (Design Tokens)**

设计令牌是构建一致性 UI 的基础，涵盖色彩、排版、间距与形状属性。开发落地时需严格遵循以下定义。

**色彩系统**

|   |   |   |
|---|---|---|
|Token Name|Hex Value|Usage Description|
|**Primary**|**#EB2F96**|品牌主色，用于主按钮、选中状态、关键强调。|
|**Text-Primary**|#333333|一级标题、正文主要内容。|
|**Background**|#F5F5F5|页面底色，营造层级感。|
|**Border**|#E0E0E0|分割线、未选中边框。|
|**Warning**|#FFA940|警告状态、待处理提示。|

**10.** **关键流程与数据策略**

**首页浏览：** 用户浏览分镜列表，状态圆点提供即时反馈。

**详情编辑：** 点击卡片进入详情，数据本地缓存，支持离线编辑。

**参数配置：** 全局或单镜参数调整，自动保存修改。

**一致性检查：** 系统后台自动运行检查，前台展示红/黄警示。

**导出交付：** 最终打包，若有未解决的 “错误” 级问题，弹出二次确认模态框。

|   |
|---|
|**异常与恢复策略**<br><br>· **保存冲突：** 多端同步冲突时，保留本地副本并提示用户手动合并。<br><br>· **网络错误：** 导出失败时，保留任务队列，网络恢复后自动重试。|

**11.** **无障碍与验收标准**

为确保产品的高可用性与包容性，所有交付物需通过以下验收清单。

|   |   |
|---|---|
|\|   \|<br>\|---\|<br>\|**无障碍 (Accessibility)**<br><br>· **色彩：** 文本与背景对比度严格 ≥ 4.5:1。<br><br>· **焦点：** 支持外接键盘 Tab 键导航，焦点框清晰可见。<br><br>· **辅助功能：** 滑块控件支持音量键或点击步进微调。<br><br>· **语义化：** 仅图标按钮必须包含 `aria-label` 描述。\||\|   \|<br>\|---\|<br>\|**可用性度量 (Usability)**<br><br>· **TTI (Time to Interactive):** 页面加载 ≤ 1.5s。<br><br>· **任务步数：** 导出操作从首页起 ≤ 3 次点击。<br><br>· **错误率：** 核心流程误触率 ≤ 2%。<br><br>· **响应度：** 点击反馈 ≤ 100ms。\||
