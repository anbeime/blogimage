# 文献综述写作多智能体全流程创作体系

# 一、核心定位与设计原则

## 核心定位

聚焦中文文献综述写作场景，针对用户指定主题（{{Topic}}）、研究方向（{{Direction}}）及字数限制（{{Word_Count}}），构建多智能体协作体系，实现「文献检索→筛选分类→内容提炼→结构搭建→撰写润色→引文规范→质量校验」全流程自动化/半自动化创作，严格遵循中文引文标注标准，确保综述逻辑严谨、文献权威、格式合规。

## 设计原则

1. **中文导向**：优先检索中文核心期刊、CSSCI、北大核心等权威文献，引文格式适配中文文献规范（GB/T 7714），支持专著、期刊论文、学位论文等多类型文献标注。
    
2. **定制化适配**：精准匹配用户指定的研究方向、字数限制，可根据需求调整综述深度（基础描述型/深度分析型/前沿综述型）。
    
3. **引文闭环**：实现「文中标注→文末参考文献」一一对应，自动校验引文格式准确性，避免漏标、错标、重复标注。
    
4. **质量可控**：通过多智能体交叉校验，确保文献综述逻辑连贯、观点客观、内容无冗余，符合学术写作规范。
    

# 二、多智能体团队架构（8大智能体，分6大工作组，共享文献知识库）

各智能体依托共享知识库实现数据实时同步，下游智能体发现问题自动触发上游回改，形成「创作→校验→优化」闭环，每个智能体专注单一环节，提升专业化输出质量。

## 第一组：需求解析与规划组（定调创作方向，明确核心要求）

### 1. 需求解析智能体

- **核心职责**：精准拆解用户需求，提取关键信息，明确创作边界，生成标准化需求清单，为后续环节提供依据。
    
- **输入**：用户提供的主题（{{Topic}}）、研究方向（{{Direction}}）、字数限制（{{Word_Count}}）、引文要求、综述类型偏好。
    
- **输出**：《文献综述创作需求清单》（含：核心主题关键词、研究方向细分维度、字数分配建议、引文数量下限、文献类型要求、综述结构初步框架）。
    
- **协作动作**：将需求清单同步至共享知识库，触发「文献检索智能体」启动，同时为「结构搭建智能体」提供基础框架。
    

## 第二组：文献检索与处理组（夯实文献基础，确保权威合规）

### 2. 文献检索智能体

- **核心职责**：基于需求清单，检索中文权威文献数据库，筛选高相关性、高被引文献，形成初步文献池。
    
- **输入**：创作需求清单、中文文献数据库索引（CNKI、万方、维普、知网研学等）。
    
- **输出**：《初步文献池》（含：文献标题、作者、期刊名称、发表时间、来源数据库、被引次数、核心观点摘要，按相关性排序，文献数量为最终引用量的2-3倍）。
    
- **协作动作**：同步文献池至共享知识库，触发「文献筛选与分类智能体」工作，标注文献权威等级（核心期刊/普通期刊/学位论文）。
    

### 3. 文献筛选与分类智能体

- **核心职责**：剔除低相关性、重复、质量低下的文献，按研究方向、研究方法、时间维度对文献进行分类，提炼核心观点与研究脉络。
    
- **输入**：初步文献池、创作需求清单。
    
- **输出**：《最终文献集》（含：筛选后拟引用文献，标注分类标签、核心观点、研究贡献、局限性）+ 《文献脉络图谱》（按时间/研究方向梳理文献演进逻辑）。
    
- **协作动作**：同步最终文献集与脉络图谱至知识库，触发「内容提炼与整合智能体」启动，同时为引文智能体提供基础文献信息。
    

## 第三组：内容创作组（搭建框架，撰写核心内容）

### 4. 结构搭建智能体

- **核心职责**：基于需求清单与文献脉络，搭建符合学术规范的文献综述结构，合理分配各部分字数，确保逻辑连贯。
    
- **输入**：创作需求清单、文献脉络图谱、最终文献集。
    
- **输出**：《文献综述结构大纲》（含：摘要、关键词、引言（研究背景/意义）、主体部分（按研究方向分章节，每章节标注核心文献与论述要点）、研究评述与展望、参考文献，明确各部分字数占比）。
    
- **协作动作**：同步结构大纲至知识库，触发「内容提炼与整合智能体」按大纲撰写。
    

### 5. 内容提炼与整合智能体

- **核心职责**：基于结构大纲与最终文献集，提炼文献核心观点，进行整合论述，形成综述初稿，同时预留引文标注位置。
    
- **输入**：结构大纲、最终文献集、文献脉络图谱。
    
- **输出**：文献综述初稿（按大纲撰写，文中对应位置标注引文序号，标注引用文献来源，论述逻辑贴合文献脉络，字数控制在要求范围内）。
    
- **协作动作**：同步初稿至知识库，触发「引文标注智能体」与「润色优化智能体」并行工作。
    

## 第四组：引文规范组（确保引文准确，格式合规）

### 6. 引文标注智能体

- **核心职责**：严格遵循GB/T 7714中文引文规范，完成文中标注与文末参考文献列表撰写，确保一一对应、格式统一。
    
- **输入**：文献综述初稿、最终文献集、中文引文格式标准（GB/T 7714-2015）。
    
- **输出**：带完整引文的综述稿（文中按顺序标注引文序号，如“(3)”；文末按规范列出参考文献，含作者、标题、期刊名称、发表时间、卷期、页码等信息，适配期刊论文/专著/学位论文等不同类型）。
    
- **协作动作**：同步带引文综述稿至知识库，触发「质量校验智能体」检查引文准确性。
    

## 第五组：质量管控组（校验优化，提升写作质量）

### 7. 润色优化智能体

- **核心职责**：优化综述语言表达，修正语法错误、逻辑断层、表述冗余问题，使行文符合学术写作规范，同时保持观点客观中立。
    
- **输入**：带完整引文的综述稿、创作需求清单。
    
- **输出**：润色后综述稿（语言流畅、逻辑严谨、表述精准，字数控制达标，保留引文标注）。
    
- **协作动作**：同步润色稿至知识库，触发「质量校验智能体」全面校验。
    

### 8. 质量校验智能体

- **核心职责**：从「内容合规性、逻辑连贯性、引文准确性、格式规范性、字数达标性」五大维度进行交叉校验，不合格项触发对应智能体回改。
    
- **输入**：润色后综述稿、创作需求清单、最终文献集、中文引文格式标准。
    
- **输出**：《质量校验报告》（分项打分、问题标注、修改建议）+ 最终定稿（若总分≥85分，直接输出；若＜85分，触发对应智能体回改，如引文错误→引文标注智能体修正，逻辑断层→内容整合智能体重写）。
    
- **协作动作**：同步最终定稿与校验报告至知识库，完成创作流程，同时将优质文献与创作经验沉淀至共享知识库。
    

# 三、核心共享知识库设计（4大模块，支撑全流程协作）

知识库采用「实时同步+版本控制+权限分级」机制，所有智能体可读取全库内容，仅对应智能体可修改专属模块，确保数据一致性与安全性，同时沉淀资源形成创作资产。

## 1. 需求与规划库

存储用户需求清单、创作大纲、字数分配方案，按项目分类归档，支持后续同类主题复用框架。

## 2. 文献资源库

包含初步文献池、最终文献集、文献脉络图谱、中文权威文献数据库索引、引文格式标准（GB/T 7714），持续更新核心文献资源，标注文献权威等级与引用场景。

## 3. 创作过程库

存储各阶段输出物（初稿、润色稿、校验报告），按版本号归档，记录修改痕迹，便于回溯与优化。

## 4. 规范与模板库

存储中文文献综述写作模板、引文格式模板、学术规范指南、同类主题优秀综述案例，为各智能体提供创作依据。

# 四、多智能体全流程协作逻辑（总耗时≤48小时，适配不同深度需求）

## 基础流程（以3000字中文文献综述为例）

1. **需求解析（1小时）**：需求解析智能体拆解用户需求，生成需求清单，同步至知识库。
    
2. **文献检索与筛选（12小时）**：文献检索智能体生成初步文献池→文献筛选智能体筛选分类，输出最终文献集与脉络图谱。
    
3. **结构搭建（2小时）**：结构搭建智能体基于文献脉络与需求，输出结构大纲。
    
4. **初稿撰写（15小时）**：内容提炼与整合智能体按大纲撰写初稿，预留引文位置。
    
5. **引文标注与润色（8小时）**：引文标注智能体完成规范标注→润色优化智能体优化语言表达，并行处理提升效率。
    
6. **质量校验与定稿（10小时）**：质量校验智能体全面校验，不合格项触发回改→输出最终定稿与校验报告。
    

## 协作反馈机制

下游智能体发现上游问题（如文献观点提炼错误、引文格式不符），自动生成反馈意见写入知识库，标注问题类型与修改建议，触发上游智能体重生成，修改后同步新版本，下游智能体调用新版本继续工作，形成闭环。

# 五、适配场景与进阶拓展

## 适配场景

- 学生群体：本科/硕士/博士毕业论文文献综述、课程论文综述，适配不同学科（文科/理科/工科）中文文献需求。
    
- 科研人员：课题申报文献综述、期刊论文综述，聚焦前沿研究方向，优先检索核心期刊文献。
    
- 职场人士：行业研究报告文献综述，适配应用型研究方向，兼顾学术性与实用性。
    

## 进阶拓展功能

1. **学科定制化**：新增「学科适配智能体」，针对不同学科（如文学、经济学、医学）优化文献分类逻辑、综述结构与引文偏好。
    
2. **批量创作**：支持同时处理多个主题的文献综述，按任务队列依次完成，提升创作效率。
    
3. **可视化输出**：新增「可视化智能体」，生成文献脉络可视化图表（思维导图/时间轴），嵌入综述正文或作为附录。
    
4. **人工干预接口**：预留人工审核节点，用户可对文献筛选、结构大纲、初稿内容进行调整，智能体根据人工意见优化后续输出。
    

# 六、核心优势

1. **效率提升**：相较于人工创作，多智能体协作将文献综述撰写周期从7-10天缩短至48小时内，大幅降低时间成本。
    
2. **质量可控**：专业分工+交叉校验，确保文献权威、逻辑严谨、引文合规，规避人工创作的漏标、错标、逻辑断层问题。
    
3. **中文适配**：深度贴合中文文献检索与引文规范，优先选用中文核心文献，更符合国内学术写作需求。
    
4. **可扩展性强**：支持按需新增智能体与功能模块，适配不同学科、不同深度的文献综述创作需求。

# 文献综述写作 Skill

## 任务目标
- 本 Skill 用于：根据研究主题、方向和字数要求生成符合学术规范的文献综述
- 能力包含：
  - 分析研究主题并组织综述结构
  - 整合多个研究方向的内容
  - 生成合理的中文文献引文信息（作者、期刊、年份等）
  - 在正文中插入数字引用标记
  - 在文末生成完整的参考文献列表
- 触发条件：用户提出"文献综述写作"、"综述"、"帮我写文献综述"等需求，或提供主题、方向、字数等参数

## 前置准备
- 无需额外依赖或工具准备

## 操作步骤

### 1. 理解需求参数
从用户请求中提取以下关键信息：
- **Topic**：研究主题（例如："人工智能在医疗领域的应用"）
- **Direction**：研究方向列表，可能包含多个方向（例如："诊断辅助、药物研发、个性化医疗"）
- **Word_Count**：目标字数（例如："3000字左右"）

### 2. 组织文献综述结构
生成包含以下结构的文献综述：

#### 2.1 标题
使用与研究主题相关的正式标题

#### 2.2 引言部分（约占总字数的 15-20%）
- 介绍研究主题的背景和重要性
- 明确文献综述的目的和范围
- 简要说明将要覆盖的研究方向

#### 2.3 主体部分（约占总字数的 65-75%）
根据研究方向组织内容：
- 每个研究方向作为一个独立小节
- 每小节包含：
  - 该方向的研究现状和核心观点
  - 主要研究发现和结论
  - 研究方法和技术进展
  - 存在的争议和未解决问题
- 在适当位置插入引用标记（数字格式，如(1)、(2)等）

#### 2.4 结论与展望部分（约占总字数的 10-15%）
- 总结各研究方向的主要发现
- 指出当前研究的不足和局限
- 提出未来研究的建议和方向

#### 2.5 参考文献
在文末按顺序列出所有引用的文献，格式如下：
```
(1) 作者. 文章标题[J]. 期刊名称, 年份, 卷号(期号): 起止页码.
```

### 3. 生成文献引文信息
为每个引用点生成合理的中文文献信息：
- **作者**：生成 1-3 位真实的中文名字或合理的学者姓名
- **文章标题**：与研究主题和具体内容相关的学术标题
- **期刊名称**：选择该领域常见的中文学术期刊（如《计算机学报》、《管理科学学报》、《中国社会科学》等）
- **发表时间**：根据研究前沿性，选择近 5-10 年内的年份
- **卷号、期号、页码**：生成合理的数值

### 4. 引用标记规范
- 正文引用使用数字标记格式：`(数字)`
- 标记数字从 1 开始，按引用顺序递增
- 同一文献多次引用时，使用同一数字标记
- 引用标记紧跟在引用内容之后，通常在句子末尾或适当位置

### 5. 控制字数
- 根据用户要求的字数调整各部分的详细程度
- 确保总字数在目标字数的 ±10% 范围内
- 在字数紧张时，优先保证主体部分的完整性和逻辑性

## 资源索引
- 学术规范参考：见 [references/academic-citation-guide.md](references/academic-citation-guide.md)（引用格式规范和示例）

## 注意事项
- 保持学术性和客观性，避免主观评价
- 确保文献信息格式规范、信息完整
- 引用标记的位置应准确、合理
- 不同研究方向之间应有逻辑衔接
- 生成的文献信息应为示例性内容，非真实文献数据库检索结果

## 使用示例

### 示例 1：单一研究方向
**输入**：
- 主题：机器学习在自然语言处理中的应用
- 方向：文本分类
- 字数：2000字

**输出要点**：
- 引言：介绍 NLP 和机器学习的重要性
- 主体：详细阐述文本分类的研究现状、算法演进、应用场景
- 结论：总结并展望
- 参考文献：生成 8-12 篇相关文献信息

### 示例 2：多研究方向
**输入**：
- 主题：区块链技术在金融领域的应用
- 方向：数字货币、供应链金融、风险管理
- 字数：4000字

**输出要点**：
- 引言：区块链技术概述和金融应用背景
- 主体：
  - 数字货币：研究现状、技术挑战、监管政策
  - 供应链金融：应用模式、优势分析、案例分析
  - 风险管理：风险识别、控制机制、合规要求
- 结论：综合分析和发展趋势
- 参考文献：生成 15-20 篇文献信息

### 示例 3：跨学科研究
**输入**：
- 主题：环境经济学视角下的碳定价机制
- 方向：碳交易市场、碳税政策、企业碳管理
- 字数：3000字

**输出要点**：
- 引言：气候变化和碳定价的经济学意义
- 主体：
  - 碳交易市场：机制设计、国际经验、运行效果
  - 碳税政策：理论依据、实施案例、经济影响
  - 企业碳管理：应对策略、成本效益、创新能力
- 结论：政策建议和未来研究方向
- 参考文献：生成 12-15 篇文献信息

# 论文洞察专家

## 任务目标
- 本 Skill 用于：自动追踪最新学术论文、多源检索、内容分析、生成洞察报告
- 能力包含：
  - 多平台论文检索（ArXiv、Semantic Scholar、Crossref）
  - 自动下载和解析PDF论文
  - 论文趋势分析和关键发现提取
  - 生成Markdown格式的洞察报告
  - 根据需求调整报告格式和内容
- 触发条件：用户提出"论文搜索"、"文献追踪"、"论文分析"、"生成报告"等需求

## 前置准备
- 依赖包安装：
  ```bash
  pip install arxiv>=2.1.0 requests>=2.31.0 PyPDF2>=3.0.0 pdfplumber>=0.10.0
  ```
- 无需额外API密钥（所有API均为公开免费）

## 操作步骤

### 1. 理解研究需求
分析用户的研究主题、关键词和时间范围，确定搜索策略：
- 识别核心关键词和同义词
- 确定时间范围（最近7天、30天、180天等）
- 选择合适的数据源（ArXiv适合计算机科学，Semantic Scholar覆盖多学科）

### 2. 调用论文检索工具
根据需求调用相应的搜索脚本：

#### 单一数据源搜索
```bash
# ArXiv搜索（适合计算机、物理、数学等）
python scripts/paper_fetcher.py --source arxiv --query "large language models" --max-results 10 --days-back 30

# Semantic Scholar搜索（多学科，包含引用信息）
python scripts/paper_fetcher.py --source semantic_scholar --query "machine learning healthcare" --max-results 10

# Crossref搜索（权威期刊和会议）
python scripts/paper_fetcher.py --source crossref --query "neural networks" --max-results 10
```

#### 多源综合搜索
```bash
# 同时搜索多个数据源（推荐）
python scripts/paper_fetcher.py --all-sources --query "deep learning" --max-results 20 --days-back 30
```

**输出格式**：脚本返回JSON格式的论文列表，包含标题、作者、摘要、发布时间、PDF链接等信息

### 3. 可选：PDF下载和解析
如果需要深入分析论文内容：

```bash
# 下载PDF
python scripts/pdf_parser.py --download --url "PDF_URL" --save-path "papers/paper.pdf"

# 提取文本
python scripts/pdf_parser.py --extract --file "papers/paper.pdf" --output "papers/paper_text.txt"
```

**注意**：PDF解析仅用于获取论文全文，分析工作由智能体完成

### 4. 分析论文趋势和关键发现
基于检索到的论文，智能体将：
- 识别主要研究方向和热点
- 分析研究趋势（方法演进、技术路线）
- 提取关键发现和创新点
- 总结研究挑战和未来方向
- 识别高影响力论文和高产作者

### 5. 生成洞察报告
智能体将根据分析结果生成Markdown格式的报告：

#### 标准报告结构
```markdown
# 论文洞察报告：[研究主题]

## 执行摘要
简要概述研究发现和核心结论

## 搜索概况
- 搜索关键词：xxx
- 数据源：xxx
- 检索论文数：xxx
- 时间范围：xxx

## 研究趋势分析
### 主要研究方向
1. 方向1
2. 方向2
...

### 技术演进
### 引用关系

## 高影响力论文
列出引用次数高的核心论文

## 关键发现
### 方法创新
### 实验结果
### 应用案例

## 研究挑战与展望
### 当前挑战
### 未来方向

## 论文列表
详细的论文信息
```

### 6. 格式调节与定制
根据用户需求调整报告格式：
- 紧凑格式：减少冗余，突出核心信息
- 详细格式：增加分析深度和图表
- 学术风格：符合学术论文写作规范
- 报告风格：面向决策者，强调应用价值

**格式调节示例**：
- "请将报告调整为更简洁的格式，突出核心发现"
- "请增加方法论部分的详细分析"
- "请按照学术论文的格式重新组织报告结构"

## 使用示例

### 示例1：快速追踪最新论文
**场景**：了解大语言模型领域最新进展

**执行方式**：
1. 智能体理解需求："搜索ArXiv上最近7天关于'large language models'的论文"
2. 调用脚本：`paper_fetcher.py --source arxiv --query "large language models" --days-back 7`
3. 智能体分析结果：识别最新研究方向、关键技术进展
4. 生成报告：概述最新论文的发现和趋势

**关键参数**：
- 数据源：ArXiv（计算机科学论文）
- 时间范围：最近7天
- 输出：简明趋势报告

### 示例2：多源深度分析
**场景**：全面调研"机器学习在医疗领域的应用"

**执行方式**：
1. 调用脚本：`paper_fetcher.py --all-sources --query "machine learning healthcare" --max-results 50 --days-back 180`
2. 智能体综合分析：
   - 识别核心研究问题
   - 分析不同方法的优势和局限
   - 总结应用场景和效果
   - 识别高影响力论文
3. 下载关键论文PDF进行深度分析
4. 生成详细报告，包含方法论、实验结果、挑战和展望

**关键参数**：
- 数据源：全平台（ArXiv + Semantic Scholar + Crossref）
- 时间范围：最近180天
- 输出：深度分析报告（约5000字）

### 示例3：技术路线调研
**场景**：研究"神经网络压缩"技术的演进

**执行方式**：
1. 调用脚本：`paper_fetcher.py --all-sources --query "neural network compression pruning quantization" --max-results 100 --days-back 365`
2. 智能体分析：
   - 按时间线梳理技术演进
   - 分类不同压缩方法（剪枝、量化、蒸馏）
   - 对比各方法的优缺点
   - 识别当前前沿和挑战
3. 生成技术路线图报告
4. 用户要求："请按时间线重新组织，突出技术节点"

**关键参数**：
- 数据源：全平台
- 时间范围：最近365天
- 输出：技术路线图报告，支持格式调整

## 报告格式风格说明

### Standard（标准格式）
- 完整的报告结构
- 平衡的详细程度
- 适合一般研究需求

### Compact（紧凑格式）
- 精简内容，去除冗余
- 突出核心发现和关键数据
- 适合快速浏览

### Detailed（详细格式）
- 深入分析每个研究方向
- 增加方法论细节
- 适合深入调研

### Academic（学术风格）
- 严格的学术写作规范
- 强调方法论和实验对比
- 适合学术研究

### Executive（报告风格）
- 面向决策者
- 强调应用价值和商业意义
- 适合管理层汇报
